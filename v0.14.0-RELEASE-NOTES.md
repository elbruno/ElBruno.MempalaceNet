# v0.14.0: Phase 2B — Storage Validation & Diagnostics Framework

## 🎉 What's New

### ✨ New Features

#### 1. **IVectorFormatValidator for sqlite-vec BLOB Standardization** (Issue #25)
- **IVectorFormatValidator** interface in `MemPalace.Storage` namespace
- **SqliteVecBlobValidator** reference implementation for sqlite-vec BLOB format validation
- **ValidationResult** type with detailed error reporting
- **Support for variable embedding dimensions** (768, 1536, 3072, and custom)
- **31 comprehensive unit tests** covering valid/invalid formats, edge cases, corruption detection
- **Full XML documentation** with usage examples

**Benefits:**
- Prevents corrupted vector data from entering sqlite-vec storage
- Standardized validation pattern across MemPalace consumers (including OpenClawNet)
- Testable, reusable validation logic
- Clear error messages for debugging

**Usage Example:**
```csharp
var validator = new SqliteVecBlobValidator();
var result = validator.ValidateVector(myVector);
if (!result.IsValid)
{
    foreach (var error in result.Errors)
        Console.WriteLine($"Validation error: {error}");
}
```

---

#### 2. **PerformanceBenchmark Utilities for SLA Tracking** (Issue #24)
- **PerformanceBenchmark** class in new `MemPalace.Diagnostics` namespace
- **Percentile calculation** (P50, P95, P99, P100) with deterministic linear interpolation
- **SLA validation helpers** for OpenClawNet's 200ms enrichment budget pattern
- **Report generation** (markdown + JSON) for dashboards and compliance tracking
- **27 comprehensive unit tests** covering percentiles, SLA validation, edge cases (empty datasets, outliers)
- **Full XML documentation** with real-world usage examples

**Benefits:**
- Standardizes latency measurement and SLA compliance tracking across ecosystem
- Eliminates reimplementation of percentile calculation in each consumer
- Machine-consumable reports for automated dashboard integration
- Supports OpenClawNet patterns: semantic re-rank <100ms P95, health check <50ms P95, total <200ms

**Usage Example:**
```csharp
var benchmark = new PerformanceBenchmark();
benchmark.RecordLatency("semantic-search", TimeSpan.FromMilliseconds(45));
benchmark.RecordLatency("semantic-search", TimeSpan.FromMilliseconds(67));

var stats = benchmark.GetPercentiles("semantic-search");
bool isSLAMet = benchmark.ValidateSLA("semantic-search", TimeSpan.FromMilliseconds(100));

var report = benchmark.GenerateReport();
Console.WriteLine(report.MarkdownReport);
```

## 🏗️ Architecture & Design

### Storage Validation Layer
- Pure validation interface — no runtime dependencies on sqlite-vec
- Extensible design allows custom validators for other backends
- Variable dimension support enables multi-embedder scenarios
- Follows existing MemPalace error-handling conventions

### Diagnostics Framework
- New namespace `MemPalace.Diagnostics` for performance measurement utilities
- Deterministic percentile calculation (no randomness, reproducible results)
- Zero dependencies on Core or Storage layers — true utility layer
- Report format extensible for future dashboard integrations

## 📝 Migration Guide

No breaking changes. Existing code continues to work unchanged. New features are purely additive.

To use the new features:

1. **Install MemPalace NuGet packages** (if not already installed):
   ```bash
   dotnet add package MemPalace.Core
   dotnet add package MemPalace.Diagnostics
   dotnet add package MemPalace.Backends.Sqlite
   ```

2. **Import new namespaces**:
   ```csharp
   using MemPalace.Storage;           // For IVectorFormatValidator
   using MemPalace.Diagnostics;        // For PerformanceBenchmark
   ```

3. **Use the new APIs**:
   ```csharp
   var validator = new SqliteVecBlobValidator();
   var benchmark = new PerformanceBenchmark();
   ```
